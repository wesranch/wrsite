Image Classification Lab

Software:

If you have been using ArcGIS, it is recommended to use QGIS for this lab.

Overview:

You are provided with two Landsat images of the Portland metro and surrounding areas from two different image dates. The images are located in the shared folder of the class R drive. Your task is to classify pixels in the Landsat 8 image to match a classification schema of interest (e.g., land cover or forest type). You will use Google Earth for ground truth or “training” data in the classification workflow.

This lab is divided into three parts: 1. Unsupervised and Supervised Classification: Implement two different classification techniques and examine their advantages and disadvantages. 2. Accuracy Assessment: Compare your classification output to your training data to assess accuracy. 3. Change Detection Analysis: Apply the steps to the second Landsat image and assess landscape changes both qualitatively and quantitatively.

Unsupervised Pixel-Based Classification

  1. Load Bands into QGIS:

    • Load all bands into QGIS.
    • Build a true-color composite or create a virtual raster for reference.
    • Set the resolution to the highest possible and place input files into separate bands.

    Insert screenshot of your true-color composite in QGIS here.

  2. Create a Region of Interest (ROI):

    • Zoom in to a specific region of interest.
    • Create a new shapefile layer:
      • Go to Layer → Create Layer → New Shapefile Layer. Select polygon geometry and specify a file name.
    • Add features to your shapefile by selecting the pencil icon and drawing a box around your area of interest.
    • Right-click to stop editing, enter 1 for the ID, and save edits.

    Insert screenshot of your shapefile and drawn ROI here.

  3. Clip Raster by Mask Layer:

    • Search for Clip Raster by Mask Layer in the processing toolbox.
    • Input layers: Raw band images (red, green, blue).
    • Mask layer: The shapefile you just created.
    • Run the tool once for each input layer.

    Insert screenshot of clipped raster layers here.

  4. Run K-Means Clustering:

    • Search for K-means clustering for grids in the processing toolbox (equivalent to Iso Cluster Unsupervised Classification in ArcGIS).
    • Input “grids”: The clipped/masked bands (red, green, blue).
    • Set the number of clusters to match the classes in your key.
    • Define the number of iterations (consider experimenting with different values).
    • Press Run and export the resulting layer as a TIF.

    Insert screenshot of the clustering process and output layer here.

  5. Adjust Symbology and Validate:

    • Change the symbology of the output layer to paletted/unique values and use a random color ramp.
    • Validate classification by comparing with your true color composite. Toggle the layers to inspect.

    Insert screenshot of output with color ramp and validation process here.

Supervised Pixel-Based Classification

  1. Obtain Training Data:

    • You can generate random points in the layer bounds.
      • Input layer: The bounding box shapefile from step 2.
      • The number of points should be at least 10-15 points per class.

    Insert screenshot of random points or training points here.

  2. Label Points in Google Earth:

    • Open Google Earth and import your KML file (if using random points).
    • Label your points using your classification key based on land feature interpretation.
    • If the random points are not well-placed, manually adjust them in Google Earth.

    Insert screenshot of labeled points in Google Earth here.

  3. Export Training Data:

    • Right-click the KML layer in Google Earth and export it.
    • Import the KML file into QGIS and export it as a shapefile. Make sure the file name is clear (e.g., train_points_10classes_10obs.shp).

    Insert screenshot of shapefile with training points here.

  4. Add a Value Column:

    • Open the attribute table of your training points shapefile.
    • Add a new field to assign numerical values to your classes.

    Insert screenshot of the attribute table and new field here.

  5. Install dzetsaka Plugin:

    • Install the dzetsaka classification tool by going to Plugins → Manage and Install Plugins. Search for dzetsaka and install it.

    • In the Python Console, type:

      import pip
pip.main(['install', 'scikit-learn'])

    Insert screenshot of dzetsaka plugin installation here.

  6. Train Classifier:

    • Under the dzetsaka classification tool, select Train Algorithm.
    • Input raster: The raster you want the model to learn (e.g., true-color composite).
    • Input layer: Your training points shapefile. Set the field to the column with numerical value assignments.
    • Select the K-nearest neighbors algorithm (start with this one).
    • Set the pixel percentage for validation to 20% and specify the output model path.

    Insert screenshot of classifier training setup here.

  7. Predict Classifications:

    • After training, select Predict Model under the dzetsaka tool.
    • Input raster: The same raster you trained the model with.
    • Specify the output raster file name and ensure it’s saved as a .tif file.
